## Полная шпаргалка по Git

---

### 🚀 Начало работы

```bash
git init                        # создать новый репозиторий
git clone <ссылка>              # скачать репозиторий с GitHub
git remote add origin <ссылка> # привязать папку к GitHub
```

---

### 📋 Просмотр состояния

```bash
git status                      # что изменилось
git log --oneline               # история коммитов кратко
git log                         # история коммитов подробно
git diff                        # что изменилось в файлах
git branch                      # список веток
```

---

### 💾 Сохранение изменений

```bash
git add .                       # добавить все файлы
git add файл.txt                # добавить конкретный файл
git commit -m "Текст"           # сохранить коммит
git commit --amend -m "Текст"   # исправить последний коммит
```

---

### 🌿 Ветки

```bash
git branch новая-ветка          # создать ветку
git checkout новая-ветка        # переключиться на ветку
git checkout -b новая-ветка     # создать и сразу переключиться
git merge ветка                 # влить ветку в текущую
git branch -d ветка             # удалить ветку
git checkout main               # вернуться на главную ветку
```

---

### ☁️ Работа с GitHub

```bash
git push origin main            # отправить на GitHub
git push origin main --force    # отправить принудительно (после rebase)
git pull origin main            # забрать изменения с GitHub
git fetch                       # проверить изменения (без слияния)
```

---

### ⏪ Отмена изменений

```bash
git restore файл.txt            # отменить изменения в файле
git restore .                   # отменить все изменения
git reset --soft HEAD~1         # отменить последний коммит (файлы сохраняются)
git reset --hard HEAD~1         # отменить последний коммит (файлы удаляются) ⚠️
git revert <хеш>                # создать коммит который отменяет другой коммит
```

---

### 🔀 Перемещение по коммитам

```bash
git checkout <хеш>              # перейти к коммиту (detached HEAD)
git checkout main               # вернуться обратно на ветку
git checkout -b ветка <хеш>    # создать ветку от конкретного коммита
```

---

### ⚡ Rebase

```bash
git rebase main                 # перенести коммиты поверх main
git rebase --continue           # продолжить после исправления конфликта
git rebase --abort              # отменить rebase полностью
```

---

### 🗄️ Stash (временное хранилище)

```bash
git stash                       # спрятать изменения
git stash pop                   # достать изменения обратно
git stash list                  # список спрятанных изменений
```

---

### 🔁 Типичный рабочий процесс каждый день

```bash
git checkout main               # 1. убедись что ты на main
git pull origin main            # 2. забери последние изменения
# ... редактируй файлы ...
git add .                       # 3. добавь изменения
git commit -m "Что сделал"      # 4. сохрани коммит
git push origin main            # 5. отправь на GitHub
```

> 💡 Если следовать этому порядку каждый раз — проблем с detached HEAD и конфликтами будет намного меньше
